# StreamONER

Discord の VC 表示、YouTube ライブチャット、アバター（PNG レイヤー口パク）を **1 本の OBS ブラウザソース** にまとめられる、配信向けデスクトップアプリです。

| リンク | 内容 |
| --- | --- |
| [quick-start-3min.txt](./quick-start-3min.txt) | 最短セットアップ |
| [DEVELOPMENT_GUIDE.md](./DEVELOPMENT_GUIDE.md) | 開発者向け技術リファレンス |
| **対象 OS** | 配布インストーラは **Windows（x64）** |

---

## できること

| 機能 | 概要 |
| --- | --- |
| **Discord** | 参加中 VC のメンバー表示・発話ハイライト（RPC + OAuth） |
| **YouTube** | ライブチャット取得（InnerTube または Data API）、OBS 表示、ダッシュボードでモデレーション |
| **アバター** | 最大 2 人分の PNG レイヤー、マイク連動の口パク・母音口形・揺れ・まばたき・LookAt |
| **統合 OBS** | Discord / YouTube / アバターを `http://127.0.0.1:3000/suite` に統合 |
| **レイアウトスロット** | OBS 配置・チャット表示・アバター見た目等を最大 3 枠で保存し一括切替 |
| **OBS イベント連動** | スパチャ・メンバーシップ・キーワードで OBS ソースを一時表示/切替 |
| **セッションログ** | コメント記録、ハイライト、終了後に YouTube タイムスタンプ URL を生成 |
| **リハーサル** | モックコメント + 統合レイアウトのドラッグ編集（配信前確認） |
| **スマホダッシュボード** | 同一 LAN から PC 版と同様のリモコン操作（既定 OFF） |
| **設定バックアップ** | JSON エクスポート/インポート（シークレットは除外） |

## 必要なもの

| 項目 | 必須 |
| --- | --- |
| OBS Studio（ブラウザソース） | ○ |
| Discord Developer Portal の Client ID / Secret | Discord 利用時 |
| **Google / YouTube アカウント**（配信チャンネルと同じ） | チャット利用時（**YouTube 連携**推奨） |
| YouTube ライブの **動画 ID** | 連携しない場合のみ必須 |
| YouTube Data API キー | **不要**（アプリ同梱。上級者向け手動設定のみ任意） |
| Google Cloud プロジェクト | **不要**（開発者が OAuth / API キーを同梱） |
| マイク（A/B 最大 2 本） | アバター利用時 |

## インストールと起動

### 配布版（Windows）

1. インストーラ（`.exe`）を実行
2. **StreamONER** を起動
3. システムトレイに常駐（ウィンドウを閉じても終了しない）

### システムトレイ（右クリック）

| メニュー | 内容 |
| --- | --- |
| ダッシュボード / 設定 / リハーサル | 各ウィンドウを開く |
| チャット取得を開始/停止 | YouTube ポーラー（要確認時はダッシュボードを開く） |
| OBS URL をコピー | 統合 URL・個別 URL |
| デスクトップ透過 | 表示・クリック透過・位置ロック（サブメニュー） |
| Discord RPC を再接続 | Discord 接続のやり直し |
| 終了 | アプリ終了 |

**ダブルクリック** でダッシュボードを開きます。

## 初回セットアップ

**全般** タブのチェックリストに沿って進めるのがおすすめです。

| 順 | タブ | 内容 |
| --- | --- | --- |
| 1 | **全般** | テーマ、配信表示 ON/OFF（Discord / チャット / アバター） |
| 2 | **接続** | Discord Client ID / Secret、OAuth。**YouTube 連携**（推奨） |
| 3 | **接続** | OBS WebSocket（ダッシュボードから OBS 操作する場合） |
| 4 | **チャット** | 表示設定、フィルター（動画 ID はダッシュボード／連携） |
| 5 | **アバター** | マイク、PNG、表示人数（使う場合） |
| 6 | **レイアウト** | 統合 OBS URL を OBS に登録、パネル配置、リハーサル確認 |
| 7 | **ダッシュボード** | 配信開始後にチャット取得開始、ランプ・タイマーで配信確認 |

設定は **自動保存** されます（Client Secret・API キーはフォーカスが外れたときに保存）。

## OBS の設定（統合 URL）

| 項目 | 値 |
| --- | --- |
| URL | `http://127.0.0.1:3000/suite` |
| サイズ | **1920 × 1080**（16:9 固定） |
| ページ透過 | オン |

1. OBS でブラウザソースを追加し、上記 URL・サイズを指定
2. **レイアウト** タブの **統合 OBS レイアウト** で Discord / チャット / アバター各パネルを調整
3. **リハーサル** で配信前に見た目を確認（編集モードでドラッグも可能）

### 配信表示 OFF 時

統合 URL は約 1 秒ごとに ON/OFF を取得し、OFF の iframe を非表示にします。チャット OFF 時は取得も停止します。

### 個別 URL（上級者向け）

| 用途 | URL |
| --- | --- |
| Discord のみ | `http://127.0.0.1:3000/overlay` |
| YouTube のみ | `http://127.0.0.1:3002/overlay` |
| アバターのみ | `http://127.0.0.1:3003/overlay` |

## YouTube チャット取得

### 推奨: YouTube 連携（OAuth）

**接続** タブの **YouTube 接続** で Google アカウントと連携します。ユーザーが用意するのは **配信に使う Google / YouTube アカウント** だけです。

| 必要 | 不要 |
| --- | --- |
| Google / YouTube アカウント（配信チャンネルと同じ） | Google Cloud プロジェクト |
| 初回のみブラウザでログイン + 許可 | API キーの作成・入力 |
| 配信時に YouTube でライブ配信を **開始済み** | 毎回の動画 ID / 配信 URL のコピー |

**初回セットアップ**

1. 設定 › **接続** →「YouTube と連携」
2. ブラウザが開く → 配信に使う Google アカウントでログイン
3. 「StreamONER が YouTube データの表示を求めています」→ **許可**
4. アプリに戻る →「連携済み」と表示

**毎回の配信**

1. YouTube / OBS で配信を開始
2. ダッシュボード（またはスマホリモート）の **チャット取得 › 開始**
3. 検出された配信タイトルを確認 → 開始
4. 同時に複数ライブがある場合は一覧から選択

連携していない場合は、初回に連携を促す案内が出ます（「あとで」で抑制可）。手動の動画 ID でも従来どおり動作します。

> **ベータ配布時:** Google の OAuth 同意画面が **Testing** の場合、「未確認のアプリ」と表示されることがあります。**詳細** → 続行で利用できます（テストユーザー登録が必要な場合あり）。

### 手動フォールバック（動画 ID）

OAuth を使わない場合は **ダッシュボード** で動画 ID を入力します（ライブ URL の `?v=` 以降）。

配信時はダッシュボードの **チャット取得** で開始/停止します（表示 ON/OFF とは別操作）。

### 取得方式（詳細設定）

**チャット** タブ末尾の **「詳細設定（チャット取得 API）」** で切り替えられます。

| 方式 | 動作 |
| --- | --- |
| **自動（既定）** | API キーあり → Data API v3。なし → InnerTube。InnerTube 失敗時はキーがあればフォールバック |
| **InnerTube** | API キーがあっても InnerTube を使用（試験用） |
| **Data API v3 のみ** | Google Data API のみ（キー必須） |

API キーは **チャット** タブの「手動設定（フォールバック）」内で任意設定できます。OAuth 連携が本線のため、通常は入力不要です。InnerTube を試すだけならキーを消す必要はありません。

取得中は詳細設定内に `取得中: InnerTube` / `Data API` と表示されます。

## アバター

### 表示とマイク

- **表示人数:** 2人 / 1人目のみ / 2人目のみ
- **マイク A/B:** しきい値・感度・VU メーターで調整
- **プレビュー:** 設定から別ウィンドウで口パク・笑いをシミュレート

### レイヤー・動き（左 / 右）

| 項目 | 説明 |
| --- | --- |
| 画像 | 身体・顔・髪・目・口など PNG。母音用 `mouth-a`〜`o`、瞳 `eyes-pupil` は任意 |
| スロット位置 | キャラ全体（2 人表示時の左右寄せ） |
| 自動ゆらぎ / 遅延追従 | 髪・顔の自然な揺れ、目・口の体への追従 |
| まばたき | 最小〜最大秒の範囲で自然な間隔（三角分布） |
| LookAt | `eyes-pupil` があるとき瞳をゆっくり動かす |
| カスタム部位 | 基本部位の子として PNG を追加（親の動きに追従） |

### 位置の 3 層

| 層 | 設定場所 | 動く対象 |
| --- | --- | --- |
| 統合 iframe | **レイアウト** → アバター パネル | OBS 上の枠全体（幅・高さ・位置を指定） |
| スロット位置 | **アバター** → 表示設定 | 左/右キャラ全体（パネル幅・高さに対する %） |
| レイヤーオフセット | **アバター** → 各部位 | パーツ単位 |

スパチャ・メンバーシップの演出は **OBS イベント**（設定の OBS 連携）でシーン切替等を行います。アバターの笑顔連動は行いません。

## リハーサル

`http://127.0.0.1:3000/rehearsal`（**レイアウト** タブからも開けます）

ブラウザのみの確認は `npm run preview:ui` → `http://127.0.0.1:5190/rehearsal`

- 1920×1080 の統合レイアウトでモックコメントを自動再生
- **本番と同様**、配信表示 OFF のパネルは非表示（`/suite-flags` を 1 秒ごとに取得）
- **レイアウト編集を開始** でパネルをドラッグ・リサイズ（変更は自動保存、設定と双方向同期）
- 編集中は**操作中・選択中のパネル以外**は設定・スロット切替の変更を反映
- **設定で編集** / ダブルクリックで **レイアウト** タブの該当セクションへフォーカス
- アバター既定幅は **960px**（他パネルと同様に変更可能）

## ダッシュボード

トレイまたは設定ヘッダーから開きます。

### レイアウト

| 領域 | 内容 |
| --- | --- |
| ヘッダー左 | 設定・リハーサル |
| ヘッダー右 | 状態ランプ、配信タイマー、現在時刻 |
| 操作バー | ミュート A/B、シーン、チャット取得、OBS、配信表示 ON/OFF |
| 左ペイン | スパチャ + コメント一覧 |
| 右ペイン | ピン留め、セッションログ、参加者、NG |

### 状態ランプ

スイッチ **OFF** のときはグレー。**ON** のときのみ色で稼働状況を表示します。

| 色 | 意味 |
| --- | --- |
| 緑 | 正常稼働 |
| 黄 | 待機・一部のみ |
| 赤 | 未設定・エラー |

チャットの **表示 ON** と **取得開始** は別です。緑は「取得中」を表します。

### 配信タイマー

チャット取得の開始/停止に連動。同じ動画 ID なら再起動後も経過を継続。動画 ID を別 ID に変えると 0:00:00 にリセット。

### OBS WebSocket

**接続** タブでホスト・ポート・パスワードを保存後、ダッシュボードで接続。シーン切替・マイクミュートが可能（配信/録画ランプは表示のみ）。

### スマホダッシュボード

**全般** で有効化 → QR / LAN URL → PIN ログイン。リモコンタブとチャットタブで PC 版と同期操作。

### セッションログ

記録開始/終了、ハイライト、YouTube タイムスタンプ URL 生成。**チャット** タブでラグ補正を設定。

| OS | 例 |
| --- | --- |

## トラブルシューティング

### OBS に何も出ない

- アプリ起動中か（トレイアイコン）
- URL が `127.0.0.1:3000/suite`、サイズ 1920×1080 か
- **全般** で該当機能が OFF になっていないか
- レイアウト変更後は OBS ブラウザソースを **更新**

### Discord が接続できない

- Client ID / Secret、Developer Portal の Redirect 設定
- **再接続**、必要なら OAuth クリア後に再承認

### YouTube チャットが来ない

- YouTube で **ライブ配信を開始済み** か
- **全般** でチャット ON、ダッシュボードで **取得開始** しているか
- **連携済み** の場合: 開始時に配信が検出されたか（未配信ならエラー表示）
- **手動 ID** の場合: 動画 ID が正しいか
- 連携の有効期限切れ → **接続** タブで再連携
- **詳細設定** の取得方式を確認（InnerTube 試験中なら `取得中:` 表示を確認）
- ポーリング間隔は **5000 ms** 以上推奨

### アバターが動かない

- **全般**・**アバター** で ON か
- マイク選択・しきい値（VU で確認）
- PNG パスが有効か（プレビューで確認）

### ポート競合

| ポート | 用途 |
| --- | --- |
| 3000 | Discord HTTP（`/overlay`, `/suite`, `/rehearsal`） |
| 3001 | Discord WebSocket |
| 3002 | YouTube HTTP + WebSocket |
| 3003 | アバター HTTP + WebSocket |
| 3920 | スマホダッシュボード（有効時） |
