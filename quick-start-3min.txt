StreamONER 3分セットアップ（Windows）

1) アプリを起動
- スタートメニューから「StreamONER」を起動します。
- 起動後はトレイ常駐です（×で閉じても終了しません）。
- トレイ右クリック: ダッシュボード / 設定 / チャット取得 / OBS URL コピー など。

2) 初回設定（全般タブのチェックリストに沿って）
- 全般: 配信表示 ON/OFF（Discord / チャット / アバター）を確認。
- 接続: Discord の Client ID / Secret → 再接続で OAuth。
        YouTube は「接続」タブで連携（推奨）。未連携時はダッシュボードで動画 ID。
        OBS WebSocket（ダッシュボードから OBS 操作する場合）。
- チャット: 表示・フィルター設定（連携済みなら動画 ID は省略可）。
- アバター: 使う場合のみマイク・PNG を設定。
- レイアウト: 統合 OBS URL を OBS に登録。
        WebSocket 接続時はブラウザソースを自動検出（手動完了は「OBS に追加した」）。

3) OBS に追加（推奨）
- ソース → ブラウザ を追加
- URL: http://127.0.0.1:3000/suite
- 幅: 1920 / 高さ: 1080
- 「ページ透過を許可」: ON
- レイアウトタブでパネル位置を調整。リハーサルで配信前に確認可。
- 全般で機能 OFF → OBS 上でもその枠が消えます（約1秒で反映）。

4) ダッシュボード（配信中）
- トレイまたは設定から「ダッシュボード」を開く。
- ヘッダー右: 状態ランプ・配信タイマー（チャット取得と連動）・現在時刻。
- 操作バー: ミュート・シーン・チャット取得（開始/停止）・OBS・配信表示 ON/OFF。
- チャット: 連携済みなら配信開始後に「開始」→ 検出された配信を確認。手動 ID も可。
- チャットの「表示 ON」と「取得開始」は別操作です（ランプの緑＝取得中）。

5) うまく表示されない時
- アプリが起動中か（トレイアイコン）
- URL が 127.0.0.1:3000/suite か、サイズ 1920×1080 か
- ポート 3000～3003 を他アプリが使っていないか
- 全般で Discord / チャット / アバターが OFF になっていないか
- チャット: 連携または動画 ID 設定済みか、YouTube で配信開始後にダッシュボードで取得を開始したか
- OBS ブラウザソースを更新（再読み込み）

6) 補足
- 設定は自動保存されます。
- 保存先: %APPDATA%\stream-overlay-suite\settings.json
- 全般: データスロット・設定バックアップ。
- レイアウト: リハーサル（ドラッグ編集可）・OBS イベント連動はチャットタブ側。
- チャット取得方式の切替: チャットタブ → 詳細設定（InnerTube 試験用）。
- 二重起動時は既存ウィンドウが前面に出ます。
- 詳細は README.md を参照してください。
